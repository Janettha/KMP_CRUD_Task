🔄 Flujo de Datos

1
Usuario interactúa con la UI
Click en "Crear tarea", "Completar", o "Eliminar"
2
La UI llama al Repository
repository.addTask() o repository.toggleTaskCompletion()
3
El Repository usa el API Client
TaskApi realiza la petición HTTP al servidor
4
El Servidor procesa la petición
CRUD operation en la base de datos
5
Respuesta regresa al cliente
JSON serializado → Task object
6
Repository actualiza el StateFlow
tasks.value = nuevaLista
7
UI se actualiza automáticamente
Compose/SwiftUI reacciona al cambio de estado

💡 Código Compartido en Acción

// Este código funciona EXACTAMENTE IGUAL en Android, iOS y Web class TaskRepository { private val api = TaskApi() private val _tasks = MutableStateFlow<List<Task>>(emptyList()) val tasks: StateFlow<List<Task>> = _tasks.asStateFlow() suspend fun addTask(title: String, description: String) { val result = api.createTask(title, description) result.onSuccess { newTask -> _tasks.value = _tasks.value + newTask } } suspend fun toggleTaskCompletion(taskId: Long) { val task = _tasks.value.find { it.id == taskId } ?: return val updated = task.copy(completed = !task.completed) api.updateTask(taskId, updated) .onSuccess { updatedTask -> _tasks.value = _tasks.value.map { if (it.id == taskId) updatedTask else it } } } }

🎯 Endpoints del API

Método Endpoint Descripción
GET /tasks Obtener todas las tareas
GET /tasks/{id} Obtener una tarea específica
POST /tasks Crear una nueva tarea
PUT /tasks/{id} Actualizar una tarea existente
DELETE /tasks/{id} Eliminar una tarea
